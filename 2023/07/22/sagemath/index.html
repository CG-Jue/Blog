    记录一下sagemath的安装

    本来是装的win版的，but需要用到pwntools,只能去搞个Ubuntu的（kali寄了）

    简单记录一下win版

    win版的坑比较少，下载完有三个软件

    win使用

    依次运行sagemath、sagemath notebook、sagemath shell

    notebook运行会起一个本地的服务，里面可以存放脚本文件

    在shell中可以执行命令

    sage test.sage

    个别文件需要

    sage -python test.sage

    下载一些第三方库同样在shell

    sage --pip install XXX

    ubuntu

    坑好多

    原本是

    sudo pip install sagemath

    sudo pip install sage

    but下载完之后sage是空的

    创建一个存放的文件夹

    sudo mkdir sagemath

    sudo chmod 777 sagemath

    下载编译好的二进制文件,按自己的版本选

    下载完成后拖入Ubuntu中执行解压缩命令

    tar xvf sage-9.3-Ubuntu_18.04-x86_64.tar.bz2

    ​

    进入解压好的文件执行

    ./sage

    出现这个窗口说明运行成功

    为了方便我们构造一个软连接去全局使用命令

    先拿到当前路径

    pwd

    sudo /home/jue/桌面/sagemath/SageMath/sage /usr/loacl/bin/sage

    可以看到已经可以全局执行sage命令

    尝试一下之前的低加密指数广播攻击的sage脚本

    from Crypto.Util.number import *
from pwn import *
from tqdm import trange
n = []
c = []
for i in trange(111):
    io = remote('url', '10000')
    n.append(eval(io.recvline().decode()[3:]))
    c.append(eval(io.recvline().decode()[3:]))
    io.close()
    sleep(0.1)

e = 111
ms = crt(c, n)
R.<x> = PolynomialRing(ZZ)
f = x^e - ms
m = f.roots()[0][0]
print(long_to_bytes(m))

    sage test.sage

    会生成一个test.sage.py文件（不知道什么用）

    这里报错没有Crypto库，原本python环境是有的

    No module named 'Crypto'

    因为sage的环境和python的环境是分开的

    给sage下载和给python下载有一点区别

    我缺了三个库（下所须的）

    sahe --pip install pycryptodome --index-url https://pypi.tuna.tsinghua.edu.cn/simple
sage --pip install pwntools --index-url https://pypi.tuna.tsinghua.edu.cn/simple
sage --pip install tqdm --index-url https://pypi.tuna.tsinghua.edu.cn/simple

    下完所须库直接

    sage test.sage

    这里能运行就差不多了
